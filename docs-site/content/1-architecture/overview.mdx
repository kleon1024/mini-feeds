# Mini-Feeds 项目介绍

Mini-Feeds 是一个现代化的信息流应用，旨在提供个性化的内容推荐、搜索、广告和商品服务。本项目采用最小复杂度设计原则，在保证功能完整的同时，确保代码的可维护性和可扩展性。

<Callout type="info" icon="ℹ️">
  本教程将全面介绍如何构建一个完整的Feeds应用，从基础架构到高级功能，适合有一定开发经验的工程师学习。
</Callout>

## 技术架构

<Steps>

### 前端架构

- **框架**：Next.js (App Router)
- **语言**：TypeScript
- **样式**：TailwindCSS + shadcn/ui
- **状态管理**：React Hooks + TanStack Query
- **路由**：Next.js App Router

### 后端架构

- **框架**：FastAPI
- **ORM**：SQLAlchemy 2.0 (async)
- **迁移**：Alembic
- **验证**：Pydantic v2
- **API文档**：OpenAPI (Swagger)

### 数据存储

- **主数据库**：PostgreSQL 15+
  - 扩展：pgvector, pg_trgm, uuid-ossp
  - 数据分层：app, rel, feature, search, ads, metrics, ops
- **缓存**：Redis
  - 用途：频控、短期缓存、幂等键

</Steps>

## 项目结构

```
/app
  /backend
    /src
      /api            # FastAPI routers (/api/v1)
      /core           # config, logging, middlewares, deps
      /db             # models, schemas, migrations (Alembic)
      /services       # rec / search / ads / commerce / blend / feature / ops
      /workers        # jobs：样本、训练、物化视图刷新
    alembic.ini
  /frontend
    /app              # Next.js App Router pages
    /components
    /lib              # api client, hooks
    /styles
  /infra
    docker-compose.yml
    /sql              # init.sql、视图/物化视图/索引脚本
```

## 开发原则

1. **最小复杂度**：优先使用简单、成熟的技术方案，避免过早引入复杂组件
2. **闭环优先**：先确保基本功能闭环，再逐步增强
3. **复用优先**：尽量复用现有组件和库，减少自研
4. **可测试性**：代码设计应便于单元测试和集成测试
5. **性能优先**：关注P50延迟，优先通过缓存、索引等方式优化性能
