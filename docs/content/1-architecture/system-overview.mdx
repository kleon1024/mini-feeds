Mini-Feeds 是一个现代化的信息流应用，旨在提供个性化的内容推荐、搜索、广告和商品服务。本章节将从整体角度介绍系统架构，帮助您快速了解项目的核心组成部分。

## 系统架构概览

<Steps>

### 第一步：了解整体架构

Mini-Feeds采用前后端分离的架构，主要由以下几个核心部分组成：

- **前端应用**：基于Next.js构建的用户界面
- **后端服务**：基于FastAPI的API服务
- **数据存储**：PostgreSQL数据库和Redis缓存
- **推荐引擎**：个性化内容推荐系统
- **搜索服务**：全文检索和语义搜索
- **广告系统**：广告投放和效果评估
- **大模型应用**：LLM增强的内容生成和理解

### 第二步：理解数据流

用户与系统的交互流程：

1. 用户访问前端应用
2. 前端向后端API请求数据
3. 后端服务处理请求，调用相关服务（推荐、搜索、广告等）
4. 数据从数据库或缓存中获取
5. 后端将处理后的数据返回给前端
6. 前端渲染内容并展示给用户
7. 用户交互行为被记录并用于后续推荐优化

## 项目结构

```
/app
  /backend
    /src
      /api            # FastAPI routers (/api/v1)
      /core           # config, logging, middlewares, deps
      /db             # models, schemas, migrations (Alembic)
      /services       # rec / search / ads / commerce / blend / feature / ops
      /workers        # jobs：样本、训练、物化视图刷新
    alembic.ini
  /frontend
    /app              # Next.js App Router pages
    /components
    /lib              # api client, hooks
    /styles
  /infra
    docker-compose.yml
    /sql              # init.sql、视图/物化视图/索引脚本
```
